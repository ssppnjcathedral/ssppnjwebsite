# Auth Bug Session — April 23, 2026

## What was fixed tonight

### 1. Panel hidden behind page content (Safari z-index bug)
The nav has `backdrop-filter: blur(10px)` which creates a CSS stacking context.
In Safari/WebKit, descendants of a `backdrop-filter` parent do not reliably
render above elements outside that stacking context, even with high z-index.

**Fix:** Injected `position: fixed !important; z-index: 1500` on `#nav-mp-panel`
via `auth-avatar.js`, removing the panel from the nav's stacking context entirely.

**File:** `auth-avatar.js` → `injectStyles()`

---

### 2. Panel rendered off-screen after position:fixed
With `position: fixed`, the original CSS `top: calc(100% + 6px)` resolves to
`calc(100vh + 6px)` — below the viewport. The panel opened but was invisible.

**Fix:** Added a `MutationObserver` on the panel that fires the moment `.open`
is added to its class, then snaps `top` and `right` from the button's
`getBoundingClientRect()` before the browser repaints.

**File:** `auth-avatar.js` → `bindPanelPosition()`

---

### 3. toggleMyParish override getting clobbered
An earlier attempt overrode `window.toggleMyParish` at the top level of the
`auth-avatar.js` IIFE. But every HTML page defines `function toggleMyParish()`
in an inline `<body>` script that runs AFTER `<head>` scripts — silently
overwriting the override. Panel appeared to do nothing.

**Fix:** Moved override into `init()` (called on `DOMContentLoaded`, which fires
after all inline body scripts). Superseded by the MutationObserver approach above.

---

### 4. Flash of logged-out avatar on page navigation
`updateAvatar()` is async — it awaits `getSession()` before touching the button.
On every page load, the person-icon SVG was briefly visible before initials appeared.

**Fix:** `applyAvatarSync()` reads `sb-lmkrcjhcixlgetginvbs-auth-token` from
`localStorage` synchronously at `DOMContentLoaded`. Updates the button and
pre-populates panel content instantly, before any async call resolves.

**File:** `auth-avatar.js` → `applyAvatarSync()`

---

### 5. Panel shows as thin line (empty content)
Supabase lock contention causes `getSession()` to hang up to 5+ seconds before
resolving. `updateAvatar()` (which rebuilds panel content) was blocked behind
that await. Panel opened with no visible content — just the 1px border.

**Fix:** `applyAvatarSync()` now also pre-populates the panel with logged-in
or logged-out content directly from localStorage, so the panel has full content
the moment it opens regardless of Supabase lock status.

**File:** `auth-avatar.js` → `applyAvatarSync()`

---
